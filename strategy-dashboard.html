🔍 系統性瓶頸分析
① OI API 30天限制
Binance openInterestHist 只提供最近 30 天歷史。含 OI 條件的策略無法真正回測。

影響：8+ 策略 FAIL 根因包含此問題。
解法：付費 Coinglass API 或改用 OI-free 設計。
② 信號頻率過低
多重過濾條件（AND 邏輯）致使信號極稀少（< 5 次/年），OOS 窗口 0 交易，Sharpe=NaN。

影響：6+ 策略 FAIL。
解法：最低要求 ≥ 12 次/年，放寬部分條件或改用 OR 邏輯。
③ 移植失敗 / Pattern Decay
社群策略在原交易所有效，移植到 Binance 4H 後 alpha 消失。公開後 alpha 衰減加速。

典型案例：NFME-1H（Hyperliquid→Binance，OOS SR=-26.7）。
解法：優先研發在 Binance 數據上設計的策略。

#003 · 2026-07-01
Pipeline 第一階段複盤：4 週後無 L2 通過的系統性原因
進行中
現況（2026-07-01）：21 策略挖礦，19 已回測，1 L1 PASS，0 L2 PASS，2 PENDING。

系統性瓶頸：
① OI API 30 天限制（Binance）→ 含 OI 條件的策略無法完整回測（影響 8+ 策略）。
② 高精度多重過濾 → 信號頻率過低（<5 次/年），OOS 窗口無交易，Sharpe=NaN。
③ 移植失敗 → 社群策略在其他交易所有效，移植 Binance 後 alpha 消失（NFME OOS SR=-26.7）。

核心學習：需付費 OI 歷史（Coinglass）；或轉向不依賴 OI 的高頻信號（≥20 次/年）；或研發非反轉型策略（Session Momentum 等）。
